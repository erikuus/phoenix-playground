What is a Changeset?

As a developer, you'll use changeset all the time in Phoenix and Ecto, but it's easy to miss what it actually does. Think of a changeset as a smart container for user input: it holds the original struct (like City), any new data the user tries to submit, and a set of rules for what's valid. When you build a changeset, you tell it what fields are required, what values are allowed, and any custom checks you need. If the user enters something invalid, the changeset collects those errors so you can show helpful messages in your form. In LiveView, changesets are the bridge between your backend validation and the interactive UIâ€”they keep user input, validation, and error messages all in sync, so you never lose what the user typed, even if there are mistakes.

handle_event/3

This event handler processes form submissions:

  • If the city is successfully created using Cities.create_city/1, the new city is prepended to the list of cities in the LiveView's state using update/3, the form is reset to an empty state, and a flash message is displayed to the user.
  • If there's an error (typically a validation failure), the changeset is converted back to form state using to_form/1 to display validation errors. The form is re-rendered with the user's input and corresponding error messages, allowing users to correct mistakes without losing their input.
