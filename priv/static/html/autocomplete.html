The Autocomplete module provides an example of how to implement a native autocomplete feature using HTML5 datalist within a Phoenix LiveView. Here's the breakdown of how it works:

mount/3

This function initializes the LiveView when it's first loaded. It sets the :matches assign to an empty list, ready to hold the autocomplete suggestions.

render/1

The render function defines the HTML to be sent to the client:

  • It includes a form with an input field where users can type their query.
  • The phx-change="suggest" attribute binds the input field to a LiveView event; when the value of the input changes, the suggest event will be triggered.
  • The phx-debounce="500" attribute is used to limit the frequency of events sent to the server, waiting for 500 milliseconds of inactivity before sending the event, reducing the load on the server for rapidly changing input.
  • The list="matches" attribute links the input to a <datalist> element with the same ID, which contains the <option> elements for autocomplete.
  • Autocomplete functionality is turned off on the input to ensure that the browser's native autocomplete doesn't interfere with the datalist.

<datalist> and <option>

The <datalist> element is defined with an ID of "matches", and it contains <option> elements for each match. These are generated by iterating over the @matches assign with the <option :for={match <- @matches} value={match.name} /> expression, creating an option for each country name provided by the Countries.list_country/1 function.

handle_event/3 for "suggest"

This function handles the suggest event triggered by the input field's phx-change action:

  • The first clause matches when the query is an empty string, clearing the matches by assigning an empty list to :matches.
  • The second clause is called when there is a query string. It fetches matching countries using Countries.list_country(query) and assigns the resulting list to :matches.
